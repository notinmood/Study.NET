# 说明
## 事件是委托的特殊用法
1. 我们知道，对一些列具有相同签名的方法进行抽象得到某个委托，也就是说委托是类型，具体的方式是这个类型的实例（委托类比于class类，方法类别于通过new class得到的实例对象）。
2. 用刚才定义的委托和event关键字，就能定义一个事件。这个事件实际上是一个对象，这个对象内部包含一个订阅方法的列表（这个列表对外是透明不可见的），和订阅、取定等方法。
3. 在订阅者类中，可以通过event提供的订阅、取定方法和订阅者列表交互。

因为event生成的一个复合对象，因此不能使用赋值符号（=）,将同签名的方法赋值给事件对象；而是应该使用订阅/退订操作符（+=/-=），将同签名方法连接到event对象上。

## 事件模式与实现发布/订阅模式和观察者模式
- 事件模式是发布/订阅模式；
- 事件模式就是观察者模式。

- 事件模型中不仅内置实现了订阅者列表功能，也实现了订阅、退订的方法。
- 事件模式是实现发布/订阅模式、观察者模式的最佳实践
