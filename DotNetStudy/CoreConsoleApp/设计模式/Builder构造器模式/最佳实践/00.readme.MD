# 说明
## 使用场景
有时候一个类有很多属性，如果要生成这个类的实例的时候，要给这些属性赋值。赋值方式有两种：
- 使用构造函数传递参数，如果属性过多，传递的参数也必然非常多，最后多得让人“看不懂”。
- 得到实例之后，使用setter分别给这些属性赋值。很灵活，但可能导致对象在构建过程中处于不完整状态，可变性带来的线程安全性问题，无法保证对象的不变性。

使用builder模式恰好可以解决上述两个问题，尤其是第二个，可以前期做好准备后，最后通过明确的build方法给出目标对象实例。


## 实现思路

1. 在 Entity 类里面新建一个静态内部类(Java中可以这么实现。但C#的内部静态类，不支持实例成员，因此这里暂时使用非静态内部类。)，命名为XxxBuilder；

2. 把 Entity 类所有参数复制到 XxxBuilder，然后在 XxxBuilder 新建必须参数的构造器，其他参数使用变量名作为方法然后返回自身（this）以便形成链式调用；

3. 在 XxxBuilder 类新建一个 Build 方法返回，在内部类里面实例化和逐步构建出的 Entity 类实例，也是作为链式调用的结束；

## 思路概述
- Builder前面部分会进行一些列属性设置，是为了不断完善最后的目标对象实例（一直在完善进行中，并没有完成）。
- 但只有最后Build方法，才返回完全构建好的对象实例。
