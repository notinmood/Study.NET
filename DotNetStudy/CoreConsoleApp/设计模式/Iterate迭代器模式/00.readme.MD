# 说明：

## 应用场景
迭代器模式的本质：控制访问聚合对象中的元素。也就说如果多个聚合对象（内部聚合元素的方式可能不同，比如List、Dictionary他们内部元素的聚合方式就不同），那么我们可以通过迭代器模式，而获得对他们一致的访问方式。

## 具体实践
由于这种需求实在太过趋同，因此基本上每种开发语言都已经提供了完善的一致的对聚合对象的访问方式。
- dotNET下，可以统一使用foreach/in来迭代各种不同的聚合对象。
- Python下，for/in方式。
- Java下，也有for：方式

当然，如果处理更灵活和复杂的业务逻辑的时候，还是需要开发人员自己手动写迭代器，对不同的聚合对象进行迭代，以向外提供一致的方式方式。
