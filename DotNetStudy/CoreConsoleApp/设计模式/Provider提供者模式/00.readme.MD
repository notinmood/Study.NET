# 说明：

微软的提供者模式，要跟配置文件一起使用。也就是说，具体的提供者是通过配置文件确定的。

# 特别提示
在.NET Core和.NET 5+中，配置系统和提供者的管理方式发生了变化，因此ProvidersHelper类和InstantiateProviders方法可能不再适用。在新的框架中，通常使用依赖注入（DI）来管理和配置服务提供者。如果您需要在.NET 8中实现类似的功能，您可能需要自己编写代码来解析配置、创建服务实例，并可能使用依赖注入框架来注册这些服务。这通常涉及到读取应用程序的配置文件或使用Options模式来配置服务，然后使用DI容器（如ASP.NET Core的内置容器）来创建和管理服务的生命周期。
